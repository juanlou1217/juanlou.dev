# Polish Checklist

Use this file only when you need a second-pass editing checklist for a technical blog post.

## Angle

- Is there one main point
- Can the article be summarized in one sentence
- Does every major section support that point

## Reader fit

- Is the target reader obvious
- If the piece is for beginners, are assumptions explained early
- If commands are included, are prerequisites stated

## Structure

- Does the opening explain why the topic matters
- Are sections ordered from motivation to execution to reflection
- Are there any sections that repeat earlier ideas

## Language

- Replace vague claims with concrete statements
- Replace generic praise with specific tradeoffs
- Cut filler transitions
- Break long paragraphs that carry more than one idea

## Repo accuracy

- Check file paths
- Check route names
- Check command names
- Check feature status against current docs and code

## Tone

- Practical, not grandiose
- Warm but not sentimental
- Confident but not performative
- Helpful to new AI users without overselling AI
