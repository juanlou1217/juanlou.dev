# Traces

这里存放实施留痕、交付证据和问题复盘。

## 典型内容

- 一次功能实现的执行记录
- 一次问题排查的结论
- 一次发布后的回归说明
- 一次失败尝试的复盘

## 文件命名建议

- `TRACE_<DATE>_<TOPIC>.md`
- `POSTMORTEM_<TOPIC>.md`
- `DECISION_<TOPIC>.md`

## 推荐模板

可参考 [TEMPLATE.md](./TEMPLATE.md)。

如果这次需要记录“为什么这样选，而不是那样做”，可参考：
