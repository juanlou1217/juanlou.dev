# 发布与回归检查清单

## 基础检查

- [ ] `pnpm install`
- [ ] `pnpm build`
- [ ] `pnpm lint`

## 页面检查

- [ ] 首页可正常访问
- [ ] 博客列表页可正常访问
- [ ] 博客分类页可正常访问
- [ ] 至少一篇文章详情页可正常访问
- [ ] 标签页可正常访问
- [ ] 关于页可正常访问
- [ ] 实验室页可正常访问
- [ ] `/feed.xml` 可正常访问

## 内容检查

- [ ] 新增或修改的 MDX 文章可被 Contentlayer 正常解析
- [ ] 标签统计正常生成
- [ ] 搜索索引正常生成

## API 检查

- [ ] `/api/stats` 参数校验正常
- [ ] `/api/github` 在提供 `repo` 时返回预期数据

## 数据库检查

- [ ] Prisma schema 与代码保持一致
- [ ] 数据库迁移已执行或明确说明无需迁移

## 部署检查

- [ ] 环境变量已配置
- [ ] 目标平台的 Node.js 与 pnpm 版本符合要求
- [ ] 构建产物符合预期

## 记录要求

如果这次发布包含关键改动，建议在 `docs/harness/traces/` 新增对应记录，说明：

- 改了什么
- 怎么验证的
- 还剩什么风险

## 关闭能力提醒

当前默认不应作为发布回归主路径检查的能力：

- `/projects`
- 邮件 newsletter 订阅
- 复杂 lab demos
